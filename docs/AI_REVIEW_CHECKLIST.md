# AI_REVIEW_CHECKLIST.md

## Purpose

Use this checklist before accepting or committing AI-assisted work.

## File Safety

- only assigned files were changed
- no unrelated shared files were changed
- no random new files were created without a reason

## Scope Safety

- no extra features were added
- no route/model/page was invented outside the repo plan
- scaffold placeholders were preserved where they should remain

## Simplicity Check

- code is readable by a student teammate
- names are understandable
- comments are practical
- the solution is not too abstract

## Explainability Check

I can explain:

- why this file exists
- what changed
- what is still TODO
- why this is not overbuilt

## Repo Alignment Check

- matches `docs/ASSIGNMENT_MAP.md`
- matches `docs/API_CONTRACT.md` if applicable
- matches current local repo structure
- does not conflict with the weekly TODO file

## Before Commit

- update `docs/IMPLEMENTATION_STATUS.md` if needed
- update weekly TODO if needed
- leave blocker notes if incomplete
- ask for review if a shared file changed
